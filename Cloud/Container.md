# 컨테이너

## Monolithic
모놀리식 아키텍처 - 전통적인 아키텍처

서비스가 하나의 애플리케이션으로 돌아가는 구조이며 배포가 간단하다.

단점
- 스케일링 하려면 기존의 애플리케이션을 그대로 복제해야함
- 각각의 서비스에 필요한 라이브러리 버전이 다를경우 충돌문제 발생
- 조금만 수정해도 전체 빌드 및 배포필요
  

## 마이크로 서비스

모놀리식 아키텍처의 대안으로 나온 개념

장점

- 개발자가 특정 비즈니스 로직에 대해서만 집중하여 개발이 가능하다.
- 배포가 쉽다.
- 서비스단위로 스케일링이 가능하여 불필요한 서비스는 줄이고, 더 많은 자원이 필요한 서비스는 확장가능

소규모와 대규모 등 여러 분야에서 사용 중

DevOps의 역량이 상당히 중요

컨테이너란 가상머신을 사용해 마이크로 서비스를 격리하는 기술
